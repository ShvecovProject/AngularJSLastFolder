Система сборки должна поддерживать три отдельные процедуры сборки.

1) Команда быстрой проверки корректности кода: в проектах на js под этим может подразумеваться выполнение проверок с помо
щью jslint/jshint и модульных тестов.
2) Комана развертывания готового приложения для последующего тестирования. После выполнения данной команды должна появляться
возможность запустить приложение в браузере. Эта процедура занимает больше времени так как в проуессе ее выполнения
необходимо сгенерировать файлы CSS и выполнить ряд дополнительных операций.
3) Команда подготовки готового приложения к развертыванию в эксплуатационном окружении должна производить все проверки
перечисленные выше а также подготавливать приложение к окончательному развертыванию, выполняя слияние, модификацию файлов
производя интеграционное тестирование и тд.

Организация файлов и каталогов.
                                Каталоги верхнего уровня
1)Исходный код приложения и сопроводительных тестов должен храниться в разных каталогах. Это упростит сопровождение системы сборки
потому что тестирование и сбора произволяться различными комплексами задач.
2)Код любых сторонних библеотек должен храниться отдельно от внутреннего кода проекта.
3) Результаты выполнения сборки должны сохраняться в специально выделенном каталоге. а не в каталоге с исходным кодом.
Состав и содержимое файлов получившихся после сборки должны в точности соответствовать требованием среды развертывания приложения.
src: - хранит исходный код приложения
test: - хранит код автоматически выполняемых тестов.
vendor: - хранит сторонее зависимости
build:  -  хранит сценарии сборки
dist: - хранит результаты сборки, готовые к развёртыванию в целевой среде.

.gitignore: - включает правила, определяющие какие файлы не должны сохраняться в репозитории git
LICENSE: - содержит положения лицензии MIT
Grintfile.js - точка входа в систему сборки grunt js
package.json - содержит описание пакета пля приложений node.js

                                    Каталог с исходным кодом
index.html - точка входа в приложение
app, common - храниться код используемый angular JS
assets, less - код артефактов назависимый от angular JS
assets - изображения и ярлыки
less - различных переменных шаблона LESS.
